Task 1 P: Machine Learning Model Development¶

Dataset: Merged Iganga Mental Health Longitudinal Survey (Waves 1, 2 & 3)

This notebook demonstrates the full machine learning pipeline from data preprocessing through model selection and evaluation — using a mental health survey dataset collected across three waves in the Iganga district of eastern Uganda. The primary goal is to predict whether an individual has a common mental disorder (anxiety and/or depression) based on demographic, socio-economic, and psychosocial features.

1. Understanding the ML Pipeline¶

The machine learning pipeline is a structured sequence of steps that transforms raw data into a trained, validated predictive model. In practice it consists of three broad stages:

1.1 Data Preprocessing¶

Raw survey data nearly always has issues that must be resolved before modelling:

  • Handling missing values: Observations with blank or null entries can be dropped (if few) or imputed using mean, median, or mode strategies depending on the variable type. In longitudinal datasets like ours, attrition between waves is common.
  • Encoding categorical variables: Machine learning algorithms require numerical input. Categorical columns such as gender, marital status, and education level are converted via label encoding (ordinal categories) or one-hot encoding (nominal categories).
  • Feature scaling: Algorithms like Logistic Regression and SVM are sensitive to the magnitude of features. StandardScaler (z-score) or MinMaxScaler can be applied to bring all features onto a comparable scale.

1.2 Model Selection¶

Choosing the right model depends on the nature of the problem:

  • Supervised learning is used when we have labelled outcomes. For a binary target like "has mental disorder: Yes/No," classification models such as Logistic Regression, Random Forest, and Gradient Boosting are appropriate.
  • Unsupervised learning (e.g., K-Means clustering) is used for exploratory grouping when no labels exist. For instance, segmenting respondents by symptom profiles.

Key factors influencing model choice include dataset size, number of features, class balance, interpretability requirements, and computational cost.

1.3 Model Evaluation¶

After training, we need to measure how well the model generalises:

  • Accuracy: Proportion of all predictions that are correct, this is useful when classes are balanced.
  • Precision: Of all positive predictions, how many are truly positive, this is important when false positives are costly.
  • Recall (Sensitivity): Of all actual positives, how many were detected , this is critical in health screening where missing a case is critical.
  • F1-score: Harmonic mean of precision and recall, balancing both concerns.
  • ROC-AUC: Area under the Receiver Operating Characteristic curve; quantifies the model's ability to distinguish between classes across all thresholds.

8. Model Building¶

8.1 Model Selection Justification¶

We will train and compare four classification models:

  1. Logistic Regression — a simple, interpretable baseline for binary classification.
  2. Random Forest — an ensemble of decision trees that handles non-linear relationships and mixed feature types well.
  3. Gradient Boosting — a sequential ensemble method that often achieves high accuracy by learning from previous errors.
  4. Support Vector Machine (SVM) — effective in high-dimensional spaces with clear margins of separation.

Key factors influencing model selection here:

  • The target is binary (disorder vs. no disorder), so classification algorithms are appropriate.
  • The dataset has a moderate class imbalance (~67% No vs ~33% Yes), so we use class_weight='balanced' where available.
  • We have a mix of continuous and categorical (now encoded) features.
  • Interpretability matters in a health context, favouring Logistic Regression and Random Forest.

Logistic Regression and SVM both prioritised balanced detection, while Random Forest and Gradient Boosting were good at overall accuracy at the expense of almost completely ignoring the disorder class. For a mental health screening context where missing a true case is more harmful than a false alarm, SVM emerges as the strongest performer as it catches the most individuals who genuinely have anxiety or depression (44% recall) while maintaining reasonable precision.

Logistic Regression comes in as a close second with similar balance and the added advantage of being more interpretable. The tree-based models (Random Forest and Gradient Boosting), despite their higher accuracy figures, would be poor choices for deployment because a screening tool that detects only 5–8% of actual cases defeats its own purpose.

9. Model Performance and Evaluation¶

9.1 Evaluation Metrics Summary¶

Model evaluation metrics serve several purposes:

  • Accuracy gives an overall snapshot but can be misleading with imbalanced data.
  • Precision is important when we want to be confident that a predicted positive is truly positive (e.g., allocating scarce mental health resources).
  • Recall is critical in screening — we do not want to miss individuals who actually have a mental disorder.
  • F1-score balances precision and recall and is often the preferred metric when classes are imbalanced.
  • ROC-AUC summarises the model's discriminative ability across all classification thresholds.

9.4 Cross-Validation¶

A single train-test split can produce results that depend on which observations end up in each set. K-fold cross-validation addresses this by partitioning the data into K subsets, training on K-1 folds and testing on the remaining fold, then rotating through all folds. We use 5-fold Stratified Cross-Validation to maintain class proportions in each fold.

In [26]:

10.2 Justification for Model Selection¶

Based on the evaluation results, SVM emerged as the best performing model with the highest test F1-score of 0.3881 and the strongest recall of 0.4441, meaning it successfully detected 44% of individuals who actually had a mental disorder. Logistic Regression followed closely with an F1-score of 0.3545 and recall of 0.4254, providing a strong, interpretable baseline that is particularly valuable in a clinical or public-health context where model transparency matters. Both models demonstrated stable performance under 5-fold cross-validation, with mean F1-scores of 0.378 and 0.391 respectively.

In contrast, Random Forest and Gradient Boosting achieved higher overall accuracy (0.637 and 0.668) but detected only 8.1% and 5.4% of actual disorder cases, with cross-validation F1-scores of just 0.149 and 0.083. All four models produced ROC-AUC values close to 0.50, indicating limited discriminative power from the available features.

For deployment in a mental health screening context, SVM is recommended because:

It achieves the highest F1-score (0.3881) and recall (0.4441), offering the best balance between detecting true cases and minimising missed diagnoses.

In a screening application, failing to detect someone with a disorder carries greater consequences than a false alarm, making SVM's superior recall the decisive factor.

Cross-validation confirms that its performance is stable across different data splits (mean F1 = 0.378, std = 0.005).

The Random Forest model, while not the strongest classifier, still provides useful feature importance rankings showing that age, household size, and functional impairment measures are the most influential predictors.

10.3 Potential Challenges in Model Training¶

  • Class imbalance: About two-thirds of respondents do not have a disorder, which can bias models toward the majority class. We addressed this using class_weight='balanced'.
  • Ordinal vs. nominal encoding: Variables like education level have a natural ordering; one-hot encoding loses this information. Ordinal encoding could be explored as an alternative.
  • Self-reported data: Mental health screening tools like the GAD-7 and PHQ-9 rely on self-reporting, which introduces measurement noise.
  • Longitudinal attrition: Some participants drop out between waves, which may introduce selection bias if missingness is not random.
  • Feature leakage risk: We deliberately excluded the GAD-7 and PHQ-9 sub-item scores from the feature set, since they directly compose the target variable. Including them would inflate performance artificially.

11. Summary and Recommendations¶

  • The machine learning pipeline successfully demonstrates how raw survey data can be preprocessed, used to train classification models, and evaluated using standard metrics.
  • Demographic and socio-economic factors alone provide moderate predictive power for common mental disorders, suggesting that community-level screening tools could leverage basic demographic information.
  • Feature importance analysis highlights age, household size, and specific occupational and marital status categories as key predictors, consistent with existing public-health literature in low-income settings.
  • Future work could incorporate longitudinal features across waves (e.g., changes in anxiety scores between Wave 1 and Wave 2) and explore deep learning or ensemble stacking approaches for improved performance.
